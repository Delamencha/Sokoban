# Sokoban 2D

这是一个 Unity 2D 推箱子项目。项目实现了经典推箱子的基础玩法，并在此基础上加入运行时关卡编辑、关卡管理、主流程编排、关卡验证、求解器和参考解法展示等工具能力。

## 主要功能

- 经典推箱子玩法：玩家移动、推箱、墙体阻挡、箱子阻挡、胜利检测。
- 游玩辅助：撤销一步、重开关卡、上一关 / 下一关、通关弹窗、完成进度记录。
- 关卡列表：支持主流程关卡、未加入主流程、全部关卡筛选。
- 关卡文件管理：新建、编辑、复制、重命名、删除、导入、导出关卡。
- 主流程编辑：可配置正式游玩流程中的关卡顺序。
- 运行时关卡编辑器：支持地板、墙、空白、玩家、箱子、目标点等编辑工具。
- 画笔工具：支持常规画笔、实心矩形、空心矩形，并带矩形预览。
- 编辑器体验：支持撤销、Dirty State、保存确认、测试关卡流程。
- 关卡验证：支持基础结构检查和简易合法性验证。
- 关卡求解器：支持默认 A* 求解和 BFS 求解，可生成参考解法。
- 参考解法展示：可按步骤播放、回退、长按连续播放。

## 操作说明

游戏中的关卡名、操作提示和运行状态会显示在左上角 HUD 中。

游玩关卡时：

- `WASD` / 方向键：移动
- `Z`：撤销一步
- `R`：重开当前关卡
- `ESC`：打开菜单

参考解法展示时：

- `Space`：播放下一步
- `Z`：回退一步
- 长按 `Space` / `Z`：连续播放 / 回退
- `R`：重开展示
- `ESC`：打开参考解法菜单

## 数据与关卡

项目使用 JSON 保存关卡数据。运行时会在项目根目录或构建包同级目录下创建：

- `Level`：运行时关卡库
- `Config`：主流程配置、导入导出配置、初始化标记
- `Save`：玩家通关进度
- `Import`：手动导入关卡目录
- `Export`：导出关卡目录

关卡分为主流程关卡和非主流程关卡。主流程关卡用于正式游玩流程，非主流程关卡用于存放编辑中、测试中或尚未确认完成的关卡。这个划分用于区分已确认可玩的关卡和仍在制作过程中的关卡。

内置初始关卡放在：

```text
Assets/StreamingAssets/Level
```

首次运行时，如果外部 `Level` 目录不存在或为空，游戏会将 `StreamingAssets/Level` 中的关卡复制到外部 `Level` 目录，并尝试复制：

```text
Assets/StreamingAssets/Config/main_flow.json
```

作为初始主流程配置。

当前导入 / 导出采用固定目录 + 手动按钮流程，尚未接入运行时系统文件弹窗：

- 导入：手动将关卡 `.json` 文件复制到根目录 `Import` 文件夹，然后在游戏的关卡编辑器界面点击“导入”。
- 导出：在游戏中点击“导出”后，关卡文件会写入根目录 `Export` 文件夹，需要手动从该文件夹中取出。
- 导入成功的文件会被移动到 `Import/Imported`，导入失败的文件会被移动到 `Import/Failed`。

Git 项目中的 `LevelPack` 文件夹包含 2 个用于测试关卡导入的示例关卡文件。它们只用于验证导入流程，是否导入不影响正常游戏流程。

关卡 JSON 主要字段说明：

- `id`：关卡稳定 ID，用于主流程配置和通关进度记录。
- `displayName`：关卡显示名称。
- `createdAt`：创建时间，使用 ISO-8601 字符串。
- `width` / `height`：棋盘宽高。
- `validationStatus`：验证状态，包括 `未验证`、`无有效解`、`通过简单验证`、`有解`。
- `solutionActions`：参考解法行动序列，使用 `U` / `D` / `L` / `R` 表示上下左右移动。
- `tiles`：棋盘地形数据，每个字符串代表一行。
- `player`：玩家坐标。
- `boxes`：箱子坐标列表。
- `targets`：目标点坐标列表。

`tiles` 中的符号含义：

- `#`：墙。
- `.`：地板。
- 空格：空白区域，不可行走，游玩时不绘制。

玩家、箱子和目标点不直接写入 `tiles` 字符中，而是由 `player`、`boxes`、`targets` 坐标字段单独描述。坐标以左下角为 `(0, 0)`，`x` 向右增加，`y` 向上增加。

## 运行环境

- Unity：`2022.3.51f1c1`
- 平台：主要面向 Windows / Unity Editor 测试
- 游戏主场景：`Main`

打开方式：

1. 使用 Unity Hub 打开项目根目录。
2. 等待 Unity 完成资源导入和脚本编译。
3. 打开主场景后进入 Play Mode 测试。

## 可优化方向

- 关卡列表中更细致的筛选 / 排序功能。
- 关卡文件编辑的批量操作。
- 导入 / 导出功能接入运行时系统文件弹窗。
- 主流程编辑中列表项的可拖拽功能。
- 新游戏机制加入。该方向工作量相对较大，通常需要编辑器添加对应编辑能力，并对求解器进行重构或扩展。
